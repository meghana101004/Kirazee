Kirazee Platform Overview

Kirazee is an integrated retail business platform empowering Retail Business Owners (RBOs) to manage inventory, sales, expenses, and staff, while offering consumers a seamless way to order from local businesses (Restaurants, Grocery, Pharmacy, etc.).

🛒 For Consumers

  • Local business discovery based on location.
  • Real-time product catalog & menu browsing.
  • Multiple fulfillment: Delivery, Takeaway, Dine-in.
  • Order tracking & history.

💼 For Business Owners

  • Inventory & Purchase Management.
  • Staff Roles & Payroll.
  • Unified Order Dashboard (Online + Counter).
  • Comprehensive Sales & Financial Reports.

Delivery Partner User Flow & Roadmap

1. Executive Overview

Kirazee is a Retail Business Platform designed to empower Consumers, Retail Business Owners (RBOs), and Delivery Partners without monopolizing branding or operations. Unlike centralized marketplaces, Kirazee promotes individual retail brands. RBOs retain full control, choosing to use Kirazee-provided partners or their own internal fleet.

Core Philosophy: Business-centric delivery (hyperlocal model) where the merchant owns the customer relationship.

2. Delivery User Roles

The ecosystem follows a simplified structure with a single delivery-side role. Control resides with the RBO.

  • Delivery Partner (Rider): Responsible for pickup, fulfillment, payment collection, and customer interaction.
  • Retail Business Owner (RBO): Acts as the dispatcher, manually assigning orders to riders.

Mandatory Data Points: Pickup Lat/Long, Customer Address, Payment Method (COD/Prepaid), Live Location Sharing.

4. Order Lifecycle Management

Assignment Logic: Purely manual. RBOs select their own rider or a Kirazee partner based on availability.

5. Rider Technology

  • Platforms: Mobile App, Web Dashboard.
  • Notifications: WhatsApp, Email (No SMS).
  • Requirements: Own Smartphone.
  • Nav: Google Maps Integration.

6. Payment & Settlement

  • Modes: COD, UPI, Z.Wallet.
  • Collection: Direct by Partner.
  • Settlement: Monthly cycle.
  • Fees: No commission deduction logic yet.

7. Communication

  • Consumer: Live Tracking, Direct Calling, OTP.
  • Merchant: Route Polyline, Modification Rights.

8. SLA & Compliance

  • Acceptance: Mandatory completion once assigned.
  • Leave: Inform 3-4 days in advance.
  • Onboarding: Physical KYC & In-office training.

4. Operations & System Logs

Architecture Note: Every critical table has a corresponding _log table that tracks INSERT, UPDATE, and DELETE actions using MySQL Triggers. These logs store the full snapshot of old_data and new_data in JSON format.

Operational Tables

  • orders: Core order data, status, amounts.
  • cart: Temporary user cart items.
  • payments: Transaction logs (Razorpay).
  • rating: User reviews and ratings.
  • expenses: Operational costs (Rent, bills).

Log Tables (Trigger Based)

  • orders_log
  • inventory_log
  • expenses_log
  • purchases_log
  • bill_of_materials_log

API & Service Specifications

Authentication & Registration

Handles User Registration, OTP verification, and Login logic.

Registration Logic

  • Check if Mobile/Email exists.
  • If existing & verified: Return Conflict (409).
  • If existing & unverified: Update record & resend OTP.
  • If new: Create record (verified=0) & send OTP.

OTP Handling

  • Valid for 3 minutes.
  • Login: Checks latest unused OTP.
  • Verification: Updates is_verified=1 on success.

Business Creation Flow

A 3-step workflow for onboarding RBOs.

Section 1: Basic Info

Name, Category, Level (Master/Sub), Contact Info.

Section 2: Location

Address, Geo-coords, Features list.

Section 3: Financials

Razorpay credentials, Tax info, Payment setup.

Role Method Endpoint Purpose
Consumer POST /consumer/orders/create/ Create new order with cart items.
Consumer GET /consumer/wallet/{id}/ Get wallet balance and points.
Business POST /business/create-coupon/ Create discount coupons.
System POST /business/save-payment-data/ Webhook handler for payment success.

Operational Flows

Consumer Order & Payment Flow

1. User Browses -> Adds Items to Cart (Cart Total: ₹800)
       |
2. Create Order API (POST /orders/create) -> Status: PENDING_PAYMENT
       |
3. Initiate Payment (POST /payment/initiate) -> Get Razorpay Key & Order ID
       |
4. User Pays on Frontend (Razorpay Modal)
       |
5. Success -> Webhook Triggered (POST /payment/webhook/)
       |
6. Server Verifies Signature -> Updates Payment Table -> Updates Order Status: CONFIRMED
       |
7. RBO Receives Order -> Status: PREPARING -> READY -> DELIVERED
